# 🦷 Telegram-бот для стоматологии (интеграция с Firebase)

Бот работает с **той же Firebase базой**, что и твоё веб-приложение.
Все записи появляются в обоих местах в реальном времени.

---

## Как это работает

```
Пациент в Telegram  ──►  Бот  ──►  Firebase Firestore
                                         │
Ты в браузере  ──────────────────────────┘
(видишь всё в веб-приложении)
```

---

## ШАГ 1 — Получить токен бота

1. Открой Telegram → найди **@BotFather**
2. Отправь `/newbot`
3. Придумай название и username
4. Скопируй токен вида `123456789:AAF...`

---

## ШАГ 2 — Узнать свой Telegram ID

1. Найди **@userinfobot** в Telegram
2. Отправь любое сообщение
3. Запомни число (например `987654321`)

---

## ШАГ 3 — Скачать ключ Firebase (serviceAccountKey.json)

Это самый важный шаг для подключения к базе:

1. Открой [Firebase Console](https://console.firebase.google.com)
2. Выбери свой проект
3. Нажми **⚙️ Настройки проекта** (шестерёнка слева вверху)
4. Перейди на вкладку **Сервисные аккаунты**
5. Нажми **Создать закрытый ключ**
6. Скачается файл — переименуй его в `serviceAccountKey.json`
7. Положи этот файл в папку с ботом (рядом с `bot.py`)

⚠️ **Никому не передавай этот файл!** Это полный доступ к твоей базе.

---

## ШАГ 4 — Настроить config.py

Открой файл `.env` (создай его) и вставь:

```
BOT_TOKEN=123456789:AAF...
ADMIN_CHAT_ID=987654321
```

Или просто открой `config.py` и замени значения напрямую.

---

## ШАГ 5 — Настроить правила Firestore

В Firebase Console → Firestore → **Правила**, вставь:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Веб-приложение работает как раньше
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

(Если у тебя уже настроены правила — оставь как есть, бот использует Admin SDK и обходит правила)

---

## ШАГ 6 — Добавить поле telegramId в Firestore

Ничего делать не нужно! Бот автоматически добавит поле `telegramId`
к карточке пациента когда тот впервые напишет боту и привяжет свой аккаунт.

## ШАГ 7 — Запуск

### На своём компьютере (для теста):

```bash
# Установи Python 3.11+
python -m venv venv
source venv/bin/activate    # Mac/Linux
venv\Scripts\activate       # Windows

pip install -r requirements.txt
python bot.py
```

### На сервере Railway.app (бесплатно, 24/7):

1. Зайди на [railway.app](https://railway.app) → войди через GitHub
2. **New Project → Deploy from GitHub repo**
3. Загрузи папку бота на GitHub
4. В разделе **Variables** добавь:
   - `BOT_TOKEN` = токен
   - `ADMIN_CHAT_ID` = твой ID
5. Загрузи `serviceAccountKey.json` через **Files** в Railway
6. Укажи команду запуска: `python bot.py`

## Как пациент использует бота

### Первый запуск:
1. Пишет `/start`
2. Вводит номер телефона (который ты уже занёс в базу)
3. Бот находит его карту и привязывает Telegram
4. Если номер не найден — создаётся новая карта пациента

### Запись на приём:
1. Нажимает **📅 Записаться на приём**
2. Выбирает день → время → вид работы → подтверждает
3. Запись сразу появляется в твоём веб-приложении!

### За сутки до приёма:
Бот в 10:00 автоматически отправляет напоминание с кнопками:
- **✅ Да, приду!** → статус меняется на "confirmed", тебе приходит уведомление
- **❌ Не приду** → запись отменяется, тебе приходит уведомление

## Часто задаваемые вопросы

**Изменятся ли данные в веб-приложении?**
Да! Всё синхронизируется в реальном времени через Firebase.

**Что если пациент уже есть в базе?**
Бот найдёт его по номеру телефона и привяжет Telegram. Данные не дублируются.

**Можно изменить время напоминаний?**
Да, в `config.py` измени `REMINDER_HOUR = 10` на нужный час.

**Что если пациент не найден по телефону?**
Бот покажет список всех пациентов для выбора, или предложит создать новую карту.
